# 规则、身份与牌组

法官、行动合法性、消息路由、死亡结算和胜负条件全部由 Python 确定性执行，LLM 只负责玩家的发言与合法选项选择。

## 经典模式

`classic` 是无警长的 6–16 人简化板：

- 6–8 人为 2 狼，9–11 人为 3 狼，12–14 人为 4 狼，15–16 人为 5 狼。
- 固定包含预言家和女巫；7 人及以上加入猎人，其余为平民。
- 狼人每夜先私聊，再分别投票选择袭击目标；狼票平局由种子随机数裁决。
- 预言家每夜查验一名其他存活玩家，只得知显示阵营。
- 女巫有一瓶解药和一瓶毒药；默认同夜不能同时使用，猎人被毒死不能开枪。
- 白天依次发言和投票。最高票平局时，平票玩家辩解后重投；再次平票则无人出局。
- 狼人全部死亡则好人胜；存活狼人数大于等于存活好人数则狼人胜。
- 经典模式中，同阵营已死亡玩家仍共享阵营胜利，不计算电影奖金。

## 可配置桌规

```json
{
  "max_days": 20,
  "wolf_chat_rounds": 1,
  "witch_can_self_save": true,
  "witch_can_use_two_potions_same_night": false,
  "reveal_roles_on_death": false,
  "allow_self_vote": false,
  "last_words": true,
  "first_night_last_words": true,
  "night_death_last_words": false,
  "day_vote_last_words": true,
  "hunter_shot_last_words": false,
  "randomize_discussion_start": true,
  "randomize_seating": true
}
```

默认遗言规则：首夜死亡和白天放逐可遗言；第二夜起的夜间死亡、被猎人带走者无遗言。`last_words: false` 可整体关闭。

开局默认先打乱配置名单，再分配座位号和身份；每天从随机存活座位开始，按座位环形发言。恢复时沿用恢复点中的实际座位顺序。

## 电影模式

电影模式加入“死亡即失去个人获胜资格”的约束：基础阵营达成终局条件后，只有属于结算阵营且仍然存活的玩家才列入获胜者。固定的 100% 奖金池由存活赢家平分。

| `role_preset` | 人数与身份组成 |
| --- | --- |
| `movie_basic` | 10 人：狼人×2、村民×6、预言家、守卫 |
| `movie_crazy_fox` | 12 人：狼人×3、村民×5、预言家、灵媒师、守卫、妖狐 |
| `movie_prison_break` | 12 人：狼人×3、村民×3、预言家、灵媒师、守卫、共有者×2、狂人 |
| `movie_lovers` | 11 人：狼人×2、村民×5、预言家、灵媒师、守卫、丘比特 |
| `movie_mad_land` | 10 人：狼人、狂人×7、预言家、守卫 |

`werewolf demo --preset ...` 会自动采用正确人数；JSON 配置必须提供对应数量的玩家。

## 电影身份

- **灵媒师**：每晚得知前一天被放逐者显示为狼人侧还是村人侧。狂人、妖狐、丘比特等非狼人显示为村人侧。
- **守卫**：每晚保护一名其他存活玩家，阻止当晚狼袭；不能保护自己。保护成功只公开为平安夜。
- **狂人**：不知道狼人名单、不进入狼聊、没有夜间能力，查验显示村人侧。狼人阵营结算且本人存活时获胜；胜负中的狼人数只计算真正狼人。
- **共有者**：固定两人，开局只互相得知对方属于村人侧。
- **妖狐**：免疫狼人袭击，被预言家查验时死亡。基础阵营终局时若妖狐仍存活，妖狐取代基础阵营独自获胜。
- **丘比特/恋人**：丘比特开局指定两名不同玩家，可以选择自己。恋人保留原身份并拥有私聊；一人死亡，另一人殉情。基础阵营终局时两名恋人均存活，则恋人触发独占结算。

妖狐和恋人都是会覆盖基础阵营结果的第三方机制，内置牌组不会同时出现；自定义固定身份也禁止混用。

## 狂人板的特殊信息结构

`movie_mad_land` 只有一名狼人，但有七名不知道狼人的狂人。狂人会被预言家验为村人侧，因此：

- 默认公开身份池可以是“全员声称狂人”：真狂人如实声明，狼人混入，预言家和守卫也用它隐藏身份。
- 狂人声明没有身份区分度，也不提供机械免票；所有人仍可被投票，只能从技能冲突、行为和收益判断唯一狼人。
- “村人侧”结果只能排除唯一狼人，不能证明对方属于真正村人阵营。
- 已知假预言家可能只是狂人，不能直接等同狼人。
- 狂人假发查杀有可能误中唯一狼人，不能把假跳当成无风险动作。
- 真预言家只有村人侧结果时公开收益很低，却会把自己同时暴露给狼人和狂人。
- 守卫通常应隐藏并静默保护高价值信息角色，不应为了响应预言家声明而公开身份。

项目会为该牌组额外加载 `board_movie_mad_land` 策略技能。

## 固定身份与随机种子

`seed` 只影响程序随机性，不能固定真实 LLM 输出。`fixed_role` 仅用于测试或主持人预设，并且必须给所有玩家同时设置；读取配置的人会直接看到身份，正常游戏不应使用。

可直接运行以下真人加本地 bot 示例：

```bash
werewolf play --config examples/movie_lovers.json
werewolf play --config examples/movie_crazy_fox.json
werewolf play --config examples/movie_mad_land.json
```
